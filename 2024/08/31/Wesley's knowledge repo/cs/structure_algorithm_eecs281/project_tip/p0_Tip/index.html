getopt家族: 命令行解码

getopt

  • 能处理短选项，即那些以单个短划线开头的单字符选项（例如 -a 或 -b）
  • 使用一个字符串来定义所有有效的选项，其中每个有效的选项字符后面可以跟随一个冒号（:）来表示该选项需要一个参数

所需参数

  • argc
  • argv
  • optstring: 用一个字符串来描述每个flag是否需要额外参数
    • : 表示必须要额外参数
    • :: 表示额外参数可选

使用与返回值

一般我们用 while 进行判断是否完成读取，即 while ((opt = getopt(argc, argv, "ab:c:")) != -1)
从这里我们可以看出，getopt 函数返回一个 int，这个值是当前选项的 ASCII 码值，如果没有选项了，就返回 -1；且后续往往接一个 switch 语句，根据返回值进行不同的操作
对于额外输入的参数 会有一个全局变量optarg 进行标记, 当然如果是可选的参数，不输入会得到 optarg 指向 NULL 空地址；但是输入的参数是 char* 形式，我们如果要转换成string，需要 string arg{optarg}

getopt_long

  • 用于处理长选项，即那些以两个短划线开头的多字符选项（例如 --help 或 --version）
  • 也能处理简短的输入flag 如 -h
  • 也会返回一个 int，但是这个值是一个枚举值，用于表示选项的类型

option 结构体

option 表示的是每个 flag 的名称，是否需要额外参数，以及如何返回值的规定

一般我们设定 flag = nullptr, val 为等价返回的简写字符
一般最后要补写一个 {0, 0, 0, 0} 用作哨兵元素，标识数组的结束
longindex 用于记录当前选项在 longopts 中的索引, 只需要输入，一般不大会用，我们也可以用 long_options[longindex].name 获取字符串名称

std::vector 的使用

resize

更改vector的size和capacity，即申请空间并且完成初始化
可以通过 下标 subscript 进行访问比如 cin >> a[i]

reserve

提前预留空间，申请足够的capacity而不改变 size，即预留空间而不初始化空间
不能通过下标访问某些空间，因为可能没有申请，只能通过 push_back等进行添加访问

区别

resize可以将数组的空间扩大并且初始化新的空间的内容，而reserve可以预留空间但是没有初始化，只能通过固有的push_back函数进行添加和访问空间，而不能用cin>>a[i]的方式直接用下标访问这个地址

nosize

vector 实际上可以自动管理内存，即我们可以不断使用 push_back 直到电脑内存用尽
但是使用 reserve 方法可以有效管理空间，让运行更快
