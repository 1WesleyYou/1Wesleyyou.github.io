7. 流水线指令 pipeline

Created2024-10-23|Updated2024-10-25|cs_basic
|Post Views:

目的

可以压缩执行时间，从单循环的长周期短cpi和多循环的短周期长cpi结合得到
短周期 + 短 cpi (等于 多循环的cpi)

特点

在 multi-cycle 的电路基础上，添加一个 对应每个 stage 的寄存器, 一共四个
最大同时执行指令的次数: 5 (即 if, id, ex, mem, wb 各一个同时进行)

pipeline 问题

data hazard

不同指令在同一个时间戳下可能会对reg file 同时进行读取和写入操作，但是这可能会导致意想不到的结果

解决方案

  1. 代码主动避免
  2. 检测与贮存 (处理器主动等待)
  3. 自动修复为标准值

1. 代码主动避免

加入 noop 轮空 cycle

问题
  1. 硬件版本不同需要的 noop 数量不同 (新的硬件可能 pipeline 更长，需要的noop会变)
  2. 程序体变大很多
  3. 执行速度变慢

检测与贮存 Detect and Stall

detect

比较 regA 和 regB 与先前的 destReg 是否相同，如果相同说明存在先写入后读取的过程

stall
  1. 将当前步骤留在 fetch/decode (在 execute 之前，且二者应该保持一致停止否则会有 pc 经过但是没有写入 decode 流程的指令)
  2. 处理器插入 noop

terminology

以下四个表示 pipeline 的四个 registrer 层

  • IF: instruction fetch
  • ID: instruction decode
  • EX: excution
  • WB: write back
  • Mem: 写入内存
DAS 方法的问题
  1. cpi 在每次检测到 hazard 的时候都会变大
  2. 或许可以将上一步计算出来的值用另一个线路返回从而可以不写入而直接利用
