人和计算机的区别

据我们所知，人类的大脑是一个非常神奇的工具，它能理解很多各种各样的符号，而计算机很不幸只能理解 01 数据信号，那么，在算法中我们了解了计算机解决问题能力的强大之处，但是计算机少了很多符号的理解和处理能力，它在一般计算能力上是不是也会有一些缺点呢？
在一百年前就有很多大师提出了这样的问题，这个问题的发展和人类对于算法的定义变化是非常契合的，在早期算法定义为 “对于每一个不同的输入样例，能解决问题，能输出正确答案的工具”，但是这个说法还是不够全面
首先我们要定义 problem: 是一个事实和相应解决办法的集合，我们有判断问题 (decision problem) 其答案是 yes/no, 这是一类答案可以用 01 进行直观编码的问题，因此我们先从讨论这一类问题开始。

字母表、字符串和语言

字母表 alphabet

一个非空且有限的符号集合 (symbol / character) 是一个字母表 Σ\SigmaΣ
如果在一个问题没有说明的时候， Σ={0,1}\Sigma = \{0,1\}Σ={0,1}

字符串 string

由0个或者多个字母表中的字符组成的有限序列
空字符串写作 ϵ\epsilonϵ

字符串的表示

以下是通过 character 进行表示 string

  • aka^kak 表示长度为 k 的全 a 字符串
  • a∗a^*a∗ 表示长度大于等于 0 的字符串 （可以为空）
  • a+a^+a+ 表示长度大于等于 1 的字符串
    以下是通过 字母表（集合）表示 string
  • Σk\Sigma^kΣk 表示长度为 k 的由字母表 Σ\SigmaΣ 组成的数组
  • Σ∗\Sigma^*Σ∗ 表示所有有限长度的数组
  • Σ+\Sigma^+Σ+ 表示所有有限长度的非空数组
    以下是多元素顺序的表示
    a ∣ ba\ |\ ba ∣ b 表示由 a 和 b 单个字符组成的

语言 language

表示 string 的一个集合，但是是全字符串集合 Σ∗\Sigma^*Σ∗ 的一个子集，例如每一个 L⊆Σ∗L \subseteq \Sigma^*L⊆Σ∗
空的language 可以表述为 ∅\varnothing∅

一些概念问题

  1. 如果 language L={ϵ}L = \{\epsilon \}L={ϵ} 那么 L 算是 ∅\varnothing∅ 吗
    A: 不算，因为这个元素也是一个空字符串string,因此language 本身并不是空集
  2. 请问 a∗a^*a∗ 是否具有有限个string?
    A: 不好评价，因为 a∗a^*a∗ 的长度是有限个，因此其数量也是有限的，但是又是没有上限的，所以也是无穷个

problem 的表述

我们可以将 instance 和 solution 编码 (encode) 为string
因此我们可以将一个 problem 解码成一个 function, 写作 f:Σ∗→Σ∗f : \Sigma^* \to \Sigma^*f:Σ∗→Σ∗ 表示将一个 instace 映射到其 solution 中
尤其的，对于 decision problem, 我们可以将其视作 f:Σ∗→{No,Yes}f : \Sigma^* \to \{No, Yes\}f:Σ∗→{No,Yes}
或者我们也可以反过来将一个 decision problem 视作一个 language，即将所有经过 f 映射的到 yes 的字符串的集合 L:={x∈Σ∗:f(x)=Yes}L := \{x\in \Sigma^*: f(x) = Yes\}L:={x∈Σ∗:f(x)=Yes}
例如，问题为 x 是否为一个 palindrome? 那么我们可以用一个回文集合表示这个问题

自动有限状态确定机 Deterministic Finite Automata (DFA)

对于一个 decision problem，当我们输入为一个字符串的时候，我们怎么知道其是否为真？那么我们需要一个映射，能将这类问题正确映射到 yes / no, 因此我们需要一个映射图谱, 能接受一个 字符串的输入，并且经过内部映射得到 yes / no 的输出

定义

一个 DFA 是一个五元组 M=(Q,Σ,δ,q0,F)M = (Q, \Sigma, \delta, q_0, F)M=(Q,Σ,δ,q0​,F)

  • Q 是一个有限状态集合 state
  • Σ\SigmaΣ 是一个字母表 alphabet
  • δ\deltaδ 是一个转移函数 transition function, δ:Q×Σ→Q\delta : Q \times \Sigma \to Qδ:Q×Σ→Q
  • q0q_0q0​ 是一个初始状态 initial state
  • F 是一个接受状态集合 accepting final state

用词习惯 terminology

如果存在一个 DFA M 和一个字母表 Σ\SigmaΣ, 其中 MMM 能接受来自全字符串集合 Σ∗\Sigma^*Σ∗ 的某些元素，并且将其他映射到 False, 那么我们称 M recognize L
或者我们定义 L(M):={x∈Σ∗:M accepts x}L(M) := \{x\in \Sigma^*: M\ accepts\ x\}L(M):={x∈Σ∗:M accepts x} 也就是对一个 problem 的返回 Yes 集合
当 L(M)=LL(M) = LL(M)=L 时，我们称 M decides L, 也就是 L 的元素完全等于 （包含与被包含）L(M), 注意上面这个等号两端的 LLL 表示的含义不同

局限性

一个 DFA 是无法解决 L={xkyk}L = \{x^ky^k\}L={xkyk} 的判断的

证

令 s 表示 DFA 的状态数，那么 s 必然是一个有限的值
那么对于集合 {xsys}\{x^sy^s\}{xsys} 我们一共有 0 - s, 一共 s + 1 个 x 的数量的可能性，xi,xj,0<i<j<sx^i, x^j, 0<i<j<sxi,xj,0<i<j<s, 那么在 DFA 实现映射的时候有 state(xi)=state(xj)state(x^i) = state(x^j)state(xi)=state(xj) 那么对于后缀字符串 xiyi∈Lx^iy^i\in Lxiyi∈L, 而 xjyi∉Lx^jy^i\not\in Lxjyi​∈L 但是由于后缀了一样的映射，二者效果应该是一致的，但是根据 DFA 定义，后者返回是 false, 产生矛盾，因此 不存在 M 能 decide L
