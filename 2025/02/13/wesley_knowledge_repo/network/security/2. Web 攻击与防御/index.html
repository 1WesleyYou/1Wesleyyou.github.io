2. Web 攻击与防御

Web攻击与防御知识点大纲

1. 同源策略（Same-Origin Policy, SOP）

1.1 定义与作用

  • 核心问题: 限制不同源之间的数据访问;
    • 源的定义: scheme://domain:port（如 https://eecs388.org:443）;
    • 隔离资源: Cookies、DOM存储、JavaScript命名空间、硬件权限等;

1.2 跨域数据访问限制

  • 示例:
    • facebook.com的JavaScript无法通过$.get直接读取gmail.com的数据;
    • 加载跨域脚本（如<script src="http://gmail.com/chat.js">）仍受SOP限制，脚本执行上下文为加载页面的源;

1.3 跨域资源共享（CORS）

  • 机制: 通过预检请求（OPTIONS）和响应头Access-Control-Allow-Origin授权跨域访问;

2. Cookie安全

2.1 Cookie作用域与属性

  • 作用域规则:
    • 设置范围: 当前域或其父域（非公共后缀）;
      • 示例: login.site.com可为site.com设置Cookie，但不可为com;
    • 读取范围: 当前域及其父域;
  • 安全属性:
    • Secure: 仅通过HTTPS传输;
    • HttpOnly: 禁止JavaScript访问;
    • SameSite:
      • Strict: 禁止所有跨站请求携带Cookie;
      • Lax: 允许导航类跨站请求（如点击链接），禁止POST/iframe;

2.2 Cookie安全隐患

  • 默认行为:
    • HTTPS设置的Cookie可能通过HTTP泄露;
    • 未设置HttpOnly的Cookie可被第三方脚本读取;

3. 经典Web攻击与防御

3.1 跨站请求伪造（CSRF）

  • 攻击原理: 诱导用户浏览器向目标站点发送已认证的请求;

    • 示例:

    • 防御措施:

      • Referer验证: 检查请求来源域（隐私限制导致不可靠）;
      • CSRF令牌: 在表单中嵌入动态令牌（需绑定会话）;
      • SameSite Cookie: 设置为Lax或Strict;

3.2 SQL注入（SQLi）

  • 攻击原理: 通过未过滤输入篡改SQL语义;

    • 示例:

    • 防御措施:

      • 参数化查询: 分离SQL语句与数据（如cursor.execute(sql, [user])）;
      • 输入验证: 限制特殊字符（如单引号）;

3.3 跨站脚本攻击（XSS）

  • 类型:
    • 反射型XSS: 恶意输入通过URL反射到页面（如https://site.com/hello?name=<script>alert(1)</script>）;
    • 存储型XSS: 恶意内容持久化存储后展示给其他用户（如用户资料注入脚本）;
  • 防御措施:
    • 输入过滤与转义: 转义HTML特殊字符（如<→&lt;）;
    • 内容安全策略（CSP）: 限制脚本来源（如Content-Security-Policy: default-src 'self'）;

4. 攻击案例与影响

4.1 CSRF攻击实例

  • 银行转账: 攻击者构造自动转账请求，利用用户已登录状态完成操作;
  • 登录CSRF: 诱骗用户登录攻击者账户，劫持其数据;

4.2 XSS蠕虫案例（Samy Worm）

  • 传播机制: 通过MySpace用户资料注入自传播脚本;
  • 影响: 20小时内感染超过100万用户;

4.3 SQL注入后果

  • 数据泄露: 窃取用户表数据;
  • 数据破坏: 执行DROP TABLE users删除关键表;

5. 防御技术总结

攻击类型 防御措施
CSRF SameSite Cookie、CSRF令牌、Referer验证
SQL注入 参数化查询、输入过滤
XSS 输入转义、CSP策略、避免内联脚本
Cookie安全 Secure + HttpOnly + SameSite属性、限制作用域
