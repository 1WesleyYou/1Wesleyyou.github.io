5. Password

Created2025-03-11|Updated2025-03-12|cybersecurity
|Post Views:

网络层基础

  • 核心功能：
    • 寻址（Addressing）
    • 转发（Forwarding）：将数据包定向到正确接口（基于本地转发表）。
    • 路由（Routing）：全局计算转发表（通过路由协议）。
  • 架构分层：
    • 数据平面（Data Plane）：处理单个数据包的转发（快速、本地决策）。
    • 控制平面（Control Plane）：计算转发表的分布式算法（全局、慢速）。

IP协议核心设计

  • IP数据包结构：
    • 头部字段：
      字段名 位数 功能
      版本号（Version） 4 标识IP版本（如IPv4=4）。
      头部长度（IHL） 4 以32位字为单位的头部长度（如无选项时为5，对应20字节）。
      服务类型（ToS/DSCP） 8 定义优先级或拥塞通知（如ECN）。
      总长度（Total Length） 16 整个IP包的总字节数（最大65535字节）。
      生存时间（TTL） 8 防环路，每经过一跳减1，归零时丢弃包。
      协议（Protocol） 8 标识上层协议（如TCP=6，UDP=17）。
      头部校验和（Checksum） 16 仅校验头部完整性，每跳重新计算。
      源/目的IP地址 32×2 端到端通信的基础。
    • 分片相关字段：
      • 标识符（Identification）：唯一标记同一原始包的分片。
      • 标志位（Flags）：
        • DF（Don’t Fragment）：禁止分片（若MTU不足则丢弃并返回ICMP错误）。
        • MF（More Fragments）：标记是否为最后一个分片。
      • 偏移量（Fragment Offset）：以8字节为单位，指示分片在原始包中的位置。

分片与重组

  • 分片触发条件：数据包超过链路MTU（如IPv4默认1500字节）。
  • 重组逻辑：
    • 仅在目的端重组：遵循端到端原则，避免中间路由器的复杂状态管理。
    • 重组依据：标识符、偏移量、MF标志（判断是否收集全部分片）。
  • 示例：
    • 4000字节包分片为3个：
      分片 总长度 MF标志 偏移量（字节）
      1 1500 1 0
      2 1220 1 1480
      3 1320 0 2680

IPv6改进与设计哲学

  • 核心改动：
    • 地址扩展：128位地址（彻底解决IPv4地址耗尽问题）。
    • 简化头部：
      • 移除分片字段（分片由终端处理）。
      • 移除头部校验和（依赖上层协议校验，如TCP/UDP）。
      • 固定头部长度（无IHL字段）。
      • 新增流标签（Flow Label）：标记数据流，支持QoS。
    • 选项扩展：通过“Next Header”字段链式扩展（更灵活）。
  • 设计原则：
    • 端到端原则：将复杂功能（如分片、校验）移至终端。
    • 高效性：减少路由器处理开销（如去校验和、固定头部）。

关键对比：IPv4 vs IPv6

特性 IPv4 IPv6
地址长度 32位 128位
分片 支持路由器分片 仅终端分片
校验和 头部校验和 无（依赖上层）
头部长度 可变（含选项） 固定40字节
流标签 无 支持20位流标签

总结

  • 网络层角色：实现端到端通信的基础，核心为转发与路由。
  • IP协议核心：通过头部字段支持寻址、防环、分片、协议多路复用。
  • IPv6演进：简化设计、提升扩展性，适应未来网络需求。

IP路由器基础

  • 核心作用: 互联网基础设施的核心组件, 市场规模超1200亿美元, 主要厂商(Cisco, Huawei, Juniper)。
  • 容量定义:
    • 公式: Router capacity = N × R
    • N: 路由器端口数, R: 端口线速(如10/40/100 Gbps)。
  • 路由器类型:
    • 核心路由器: 高带宽(100Tbps聚合), 端口速率达400Gbps。
    • 边缘路由器: 中等带宽(1Tbps聚合), 端口速率100Gbps。
    • 小型企业路由器: 低带宽(<10Gbps), 端口速率1Gbps以下。

路由器内部结构

  • 控制平面: 路由/控制处理器, 负责路由决策。
  • 数据平面:
    • 输入线卡:
      • 任务: 接收包, 更新IP头(TTL, 校验等), 执行最长前缀匹配(LPM), 排队。
      • 挑战: 高速处理(100B包@200Gbps → 每4ns处理一个包), 依赖ASIC/网络处理器。
    • 交换结构: 连接输入与输出端口, 常见方式:
      • 共享内存, 总线, 交叉开关(非阻塞, 如Crossbar)。
    • 输出线卡:
      • 功能: 包分类(多维度范围匹配), 缓冲区管理(如Drop-Tail), 调度算法。

路由关键技术

  • 最长前缀匹配(LPM):
    • 目标: 根据目标IP地址查找最佳匹配端口。
    • 实现: 树状数据结构(如Trie树)优化查找效率(O(log n))。
    • 示例: 地址范围分层匹配(如110000**→Port1, 1101****→Port4)。
  • 调度算法:
    • FIFO: 简单但无法隔离流量, 易受拥塞影响。
    • 优先级调度: 高优先级队列优先服务。
    • 公平队列(FQ):
      • 近似比特级轮询, 按包截止时间排序服务。
      • 加权公平队列(WFQ): 按权重分配带宽。
    • 优缺点: FQ/WFQ提供公平性但复杂度高, FIFO简单但缺乏隔离。

路由器辅助的拥塞控制

  • 核心目标:
    • 隔离流量(公平性), 调整速率, 检测拥塞。
  • 公平性模型:
    • 最大最小公平性:
      • 公式: a_i = min(f, r_i), ∑a_i = C。
      • 示例: 总带宽10G, 需求8G/6G/2G → 分配4G/4G/2G。
    • 公平队列(FQ): 隔离流量, 避免RTT依赖, 允许灵活速率调整。
  • 显式拥塞通知(ECN):
    • 机制: 路由器通过IP包头标记拥塞(如ToS/DSCP位), 接收端通过ACK反馈。
    • 优点: 区分拥塞与丢包, 减少延迟, 易于部署(如数据中心应用)。
  • 争议点:
    • 公平性定义(按流/源/目的划分? 多跳惩罚? )。
    • 复杂性与实际部署的权衡(如FQ状态管理开销)。

总结

  • 路由器角色: 高速转发与公平调度的核心。
  • 拥塞控制演进: 从FIFO到WFQ+ECN, 平衡效率与公平。
  • 未来方向: 结合硬件加速(如ASIC), 协议优化(如RCP)提升性能。
