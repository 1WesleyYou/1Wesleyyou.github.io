REST API

Created2025-08-20|Updated2025-08-20
|Post Views:

Overview

REST API 代表 RE 表示性 State Transfer API;它是一种 API(应用程序编程接口), 允许不同系统之间通过互联网进行通信; REST API 的工作原理是在客户端和服务器之间发送请求和接收响应, 通常采用 JSON 格式

REST API 使用 HTTP 方法 (例如 GET, POST, PUT, DELETE)来定义可以对资源执行的操作;这些方法与 CRUD(创建, 读取, 更新, 删除) 作保持一致, 这些作用于通过 Web 作资源;

使用其中一种 HTTP 方法通过 Web URL 将请求从客户端发送到服务器;然后, 服务器使用请求的资源进行响应, 这些资源可以是 HTML, XML, 图像或 JSON, 其中 JSON 是现代 Web 服务最常用的格式

主要特性

  • 无状态 : 从客户端到服务器的每个请求都必须包含服务器完成请求所需的所有信息;服务器上不存储任何会话状态;
  • 客户端-服务器架构 : RESTful API 基于客户端-服务器模型, 其中客户端和服务器独立运行, 允许可扩展性;
  • 可缓存 : 可以将来自服务器的响应显式标记为可缓存或不可缓存, 以提高性能;
  • 统一接口 : REST API 遵循一组约定和约束, 例如一致的 URL 路径, 标准化的 HTTP 方法和状态代码, 以确保通信顺畅;
  • 分层系统 : REST API 可以部署在多层上, 这有助于提高可扩展性和安全性;
