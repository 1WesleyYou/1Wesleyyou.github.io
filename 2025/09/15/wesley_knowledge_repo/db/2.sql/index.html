2.sql

Created2025-09-15|Updated2025-09-16
|Post Views:
  1. 选择 (Selection)
  • 符号: S (sigma)
  • 用法: 从关系中选择满足特定逻辑条件的行(元组)的子集;
  • 条件构成: 条件是布尔组合(∧\wedge∧ 表示 AND,∨\vee∨ 表示 OR)的项;每个项的形式为 Attr1 <oper> Attr2 或 Attr <oper> value, 其中 <oper> 包括 { >>>, ≥\ge≥, =, ≤\le≤, <};
  • 语法: σ\sigmaσ condition(relation)
  • 示例:
    • S country=‘USA’ wedgewedgewedge sport=‘gymnastics’(Athlete): 从 Athlete 关系中选择国家为 ‘USA’ 且运动为 ‘gymnastics’ 的运动员;
    • 对应的SQL语句示例: SELECT * FROM Athlete WHERE sport = ‘gymnastics’ AND country = ‘USA’;
  1. 投影 (Projection)
  • 符号: pipipi
  • 用法: 从关系中选择特定的列(属性), 并删除不需要的列;此操作还会移除重复的元组, 因为经典关系模型是一个集合;
  • 语法: pipipi Attr_list(relation)
  • 示例:
    • pipipi sport, country(Athlete): 从 Athlete 关系中选择 ‘sport’ 和 ‘country’ 列;
      • 对应的SQL语句示例: SELECT sport, country FROM Athlete;
  1. 笛卡尔积 (Cross-Product / Cartesian Product)
  • 符号: ⋈\bowtie⋈
  • 用法: 允许我们将两个关系组合起来;结果是第一个关系的每一行与第二个关系的每一行组合;
  • 结果模式: 包含来自两个关系的所有字段, 名称默认继承(如果需要, 可以使用表名进行消歧);
  • 语法: Relation1 $ imes$ Relation2
  • 示例:
    - Athlete $ imes$ Olympics: 将 Athlete 关系中的每一行与 Olympics 关系中的每一行组合;
  1. 集合差 (Set-Difference)
  • 符号: -
  • 用法: 返回属于第一个关系但不属于第二个关系的元组;
  • 输入条件: 两个关系必须是并集兼容的, 即具有相同数量和类型的属性, 并且顺序相同;
  • 结果字段名: 使用第一个输入关系的字段名;
  • 语法: Relation1 - Relation2
  • 示例:
    • name - AthleteName: 返回 name 关系中的元组, 但不在 AthleteName 关系中;
    • 对应的SQL语句示例: SELECT FROM name MINUS SELECT FROM AthleteName; (在某些系统中也称为 EXCEPT);
  1. 并集 (Union)
  • 符号: cupcupcup
  • 用法: 返回属于第一个关系或属于第二个关系的元组;
  • 输入条件: 两个关系必须是并集兼容的, 即具有相同数量和类型的属性, 并且顺序相同;
  • 语法: Relation1 cupcupcup Relation2
  • 示例:
    • sigmasigmasigmacolor=‘red’ Boats cupcupcup sigmasigmasigmacolor=‘green’ Boats: 返回颜色为红色或绿色的船只;
  1. 重命名 (Rename)
  • 符号: ho hoho (rho)
  • 用法: 用于重命名关系或关系中的属性;
  • 用例:
    • 作为简写;
    • 执行自连接(Self-joins);
    • 当两个关系具有相同字段名时进行区分;
  • 语法: ho hohoAFTER(BEFORE) 或 ho hoho(AFTER, BEFORE) (重命名关系); 也可以重命名列, 如 A1.aid name1;
  • 示例:
    • ho hoho(A1, Athlete) $ imes$ ho hoho(A2, Athlete): 将 Athlete 关系重命名为 A1 和 A2, 用于自连接;
  1. 连接 (Join)
  • 符号: owtie
  • 用法: 从基本操作构建的附加操作, 非常有用;连接是选择和笛卡尔积的组合;
  • 语法: R owtiec S = sigmasigmasigmac (R $ imes$ S), 其中 c 是连接条件;
  • 示例:
    • pipipisname(sigmasigmasigmabid=103(Reserves) owtie Sailors): 查找预订了船只 #103 的水手的名字;这里隐式地使用了连接, 将 Reserves 和 Sailors 根据共享属性 sid 进行连接;
  1. 交集 (Intersection)
  • 符号: capcapcap
  • 用法: 从基本操作构建的附加操作, 非常有用;返回同时属于两个关系的元组;
  • 语法: Relation1 capcapcap Relation2
  • 示例:
    • pipipisname(Tempred capcapcap Tempgreen owtie Sailors): 查找预订了红色和绿色船只的水手的名字;Tempred 包含预订红色船只的水手 sid, Tempgreen 包含预订绿色船只的水手 sid;
  1. 除法 (Division)
  • 符号: /
  • 用法: 从基本操作构建的附加操作, 非常有用;通常用于处理"找到所有满足某个条件的实体"这类查询;例如, “找到预订了所有船只的水手”;
  • 定义: 如果关系 A 有两个字段 x 和 y, 关系 B 只有一个字段 y, 那么 A/B = { | forallforallforallininin B, <x, y>inininA};这意味着结果是所有这样的 x 元组(例如水手), 对于 B 中的每一个 y 元组(例如船只), A 中都存在一个 <x,y> 元组;
  • 语法: RelationA / RelationB
  • 示例:
    • ho hohoTempsids(pipipisid,bid(Reserves) / pipipibid(Boats)) owtie Sailors: 查找预订了所有船只的水手的名字;
      理解这些算符对于理解SQL查询处理至关重要;
